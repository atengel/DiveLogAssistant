# Example Queries for AIDiveLog

This document contains example questions you can ask the dive log assistant. These queries demonstrate the various capabilities of the AI-powered dive log system.

## Search Queries

### 1. Location-Based Search
**Query:** "What wreck dives did I do in Thailand?"

This query searches for wreck dives in Thailand, demonstrating location-based filtering combined with dive type filtering.

---

### 2. Wildlife and Highlights Search
**Query:** "Find all dives where I saw manta rays or whale sharks"

This query uses full-text search to find dives based on specific marine life mentioned in the highlights or content fields.

---

### 3. Depth-Based Filtering
**Query:** "Show me all my shallow dives under 20 meters"

This query filters dives by maximum depth, useful for finding recreational or beginner-friendly dive sites.

---

### 4. Complex Multi-Criteria Search
**Query:** "What cave dives did I do in Mexico that were deeper than 30 meters?"

This query combines multiple filters: dive type (cave), location (Mexico), and depth constraint (deeper than 30m).

---

## Aggregation and Analysis Queries

### 5. Statistical Questions
**Query:** "What's my deepest dive and how many total dives have I logged?"

This query requires retrieving all dives and calculating statistics like maximum depth and total count.

---

### 6. Location Frequency Analysis
**Query:** "Where did I dive most frequently? Show me my top 5 dive locations."

This query analyzes dive frequency by location and provides a ranked list of most-visited sites.

---

### 7. Time-Based Analysis
**Query:** "What's my total dive time across all my logged dives?"

This query aggregates the duration of all dives to calculate total time spent underwater.

---

## Dive Log Creation

### 8. Creating a New Dive Entry
**Query:** "I did a dive today at Blue Corner in Palau. It was a 45-minute recreational dive to 30 meters. We saw barracuda schools and reef sharks."

This query demonstrates how to add a new dive log entry using natural language. The assistant will extract all relevant information and create a structured entry.

---

### 9. Creating a Night Dive Entry
**Query:** "Add a night dive from yesterday at 8pm on the house reef in Bonaire. It was 48 minutes, max depth 20 meters, average 15 meters. We saw octopuses and bioluminescent plankton."

This query shows how to create a dive entry with specific time, location details, and highlights.

---

## Preference and Configuration

### 10. Setting User Preferences
**Query:** "My name is Alex and I prefer metric units"

This query sets user preferences that will be remembered across all future conversations. The assistant will use these preferences when displaying information.

---

## Tips for Best Results

- **Be specific**: Include location names, dive types, or depth constraints when searching
- **Use natural language**: The assistant understands conversational queries, so you don't need to use exact keywords
- **Combine criteria**: You can combine location, dive type, depth, and wildlife in a single query
- **Ask for statistics**: The assistant can calculate totals, averages, maximums, and other aggregations
- **Describe dives naturally**: When adding new dives, describe them as you would to a friend - the assistant will extract the relevant details
